# /mirror: Protocolo Espejo (PROT-001)

Este workflow implementa el **Protocolo Espejo (PROT-001)** para garantizar que las tareas de documentación se ejecuten con fidelidad absoluta, cero síntesis y ninguna pérdida de información.

### Cuándo Usar
- Al crear Especificaciones del Sistema (FFRS).
- Cuando el usuario solicita una "copia espejo" o "extracción íntegra".
- Cuando la lógica documentada, metáforas o metadatos son críticos para la ingeniería del sistema.

### Pasos

1.  **Activar Modo Espejo**:
    - Desactiva todos los filtros de "síntesis profesional" u "optimización de legibilidad".
    - Comprométete con un mapeo literal de los elementos de entrada a la estructura de salida.

2.  **Verificar Integridad de Datos**:
    - Audita el 100% de sustantivos, adjetivos y descriptores de estado (ej: activo, obsoleto).
    - Asegura que se preserven los términos metafóricos (ej: "Cuerdas", "Átomos").

3.  **Cross-Check de Registro**:
    - Si el ítem es un componente del sistema, vincúlalo a su entrada correspondiente en los Knowledge Items o la base de datos Titan 5.3.

4.  **Auditoría Forense Final**:
    - Compara el documento final línea por línea contra la fuente.
    - Si se detecta cualquier desviación por síntesis, reconstruye la sección afectada literalmente.
